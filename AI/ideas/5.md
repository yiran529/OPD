## 实验设计

### 模型

采用 **Qwen/Qwen3.5-0.8B** 作为底座模型。该模型是 24 层 hybrid 架构，hidden layout 为 `6 × (3 × (Gated DeltaNet → FFN) → 1 × (Gated Attention → FFN))`；模型卡同时将 0.8B 明确定位为 prototyping、task-specific fine-tuning 和 research 用途。([Hugging Face][1])

### 训练数据

主训练集采用 **OpenThoughts-114k-math**，数据集页面为 Hugging Face 上的 `open-r1/OpenThoughts-114k-math`。该数据集当前公开为单一 `train` split，约 **89.1k** 条样本，字段包含 `problem`、`solution`、`messages`、`conversations`、`generated_token_count`、`correct` 等，适合直接抽取题目与推理文本。([Hugging Face][2])

备选训练集保留两套：

* **GSM8K**：更短、更干净，适合 sanity check。
* **Bespoke-Stratos-17k**：可作为更一般 CoT 数据的补充。
  这两套不作为主实验，只在需要做小规模对照时启用。此前 OPSD 也是在 **OpenThoughts 的数学子集**上训练，最多采样 **30K** 条带 CoT 的 problem-solution pairs。([arXiv][3])

### 样本格式

每条样本写成：
[
x = \text{problem}, \qquad y = \text{solution}
]

其中 `solution` 作为完整 gold reasoning trajectory。初版不再额外拆 answer-only 目标，直接在 `solution` token 序列上做分布蒸馏。

### teacher / student

* **teacher**：冻结的初始 checkpoint，参数记为 (\theta_T)
* **student**：当前训练中的模型，参数记为 (\theta_S)

teacher 始终吃 **clean prefix**：
[
q_t = p_{\theta_T}(\cdot \mid x, y_{<t})
]

student 吃 **corrupted prefix**，然后在该 prefix 上继续 rollout：
[
p_t = p_{\theta_S}(\cdot \mid x, \tilde y_{<t})
]

teacher 固定为初始策略，与 OPSD 的实现一致；论文中也明确说明这种做法有助于稳定训练并起到正则化作用。([arXiv][3])

### corruption 方案

只污染 `solution`，不污染 `problem`。
对每条样本：

1. 在 `solution` 前半段到中段采样一个起点 (s)。
2. 取一个短 span，长度 (m\in{2,4})。
3. 将这段 span 替换为**同一样本内其他位置随机抽取的 token**，构造 corrupted prefix (\tilde y)。
4. 从 (s+m) 开始，让 student 在 corrupted prefix 上 **greedy rollout** 长度 (K)。

初版默认：
[
m=2,\qquad K=8
]
并用 ((m=4))、((K=16)) 做小规模消融。

### 损失函数

不使用 CE，只保留蒸馏损失。
在 student rollout 段上定义

[
L_{\mathrm{KD}}
===============

\frac{1}{K}
\sum_{t=s+m}^{s+m+K-1}
\Big[
\alpha, \mathrm{KL}(q_t ,|, p_t)
+
(1-\alpha),\mathrm{KL}(p_t ,|, q_t)
\Big]
]

其中：

* (q_t = p_{\theta_T}(\cdot \mid x, y_{<t}))
* (p_t = p_{\theta_S}(\cdot \mid x, \tilde y_{<t}))
* (\alpha \in [0,1]) 为 forward / reverse KL 的混合系数，是核心超参数

即：
[
L = L_{\mathrm{KD}}
]

我将 (\alpha) 作为主消融轴，至少测试：
[
\alpha \in {1.0,\ 0.75,\ 0.5,\ 0.25,\ 0.0}
]

其中：

* (\alpha=1)：纯 forward KL
* (\alpha=0)：纯 reverse KL
* (\alpha=0.5)：对称混合

之所以保留 pure forward KL 作为重要对照，是因为 OPSD 的消融中，**forward KL 明显优于 reverse KL 和 JSD**，并被作者选为后续实验默认目标。([arXiv][3])

### 训练细节

* teacher 冻结，student 更新
* 每条样本每步只做一次 corruption
* student rollout 使用 greedy decoding
* 蒸馏只在 rollout 的 (K) 个位置上计算
* 实现上采用 full-vocabulary logit distillation，而不是 sampled-token policy-gradient 式目标；OPSD 的正式实验同样采用 full-vocabulary logit distillation。([arXiv][3])

### 评测

#### 1. 主评测：corruption robustness

在 held-out math 样本上固定同样的 corruption protocol，报告：

* clean prefix 下的解题准确率
* corrupted prefix 下的解题准确率
* recovery gap = clean acc − corrupted acc
* rollout 段平均
  [
  \mathrm{KL}(q_t|p_t),\quad \mathrm{KL}(p_t|q_t)
  ]
  以及混合损失 (L_{\mathrm{KD}})

这一组指标是主指标，用来直接验证“是否学会纠偏”。

#### 2. benchmark

* **GSM8K**：衡量较短数学推理上的泛化
* **MATH-500**：衡量更难的数学推理
* **MMLU-Pro**：检查是否出现一般能力迁移
* **IFEval**：检查训练是否损伤指令遵循能力

Qwen3.5-0.8B 模型卡本身提供了 thinking-mode 下的 MMLU-Pro 和 IFEval 等指标，因此这两个 benchmark 适合作为通用能力副作用检查。([Hugging Face][1])

### 最小实验矩阵

只跑四组：

* **A**: (\alpha=1.0)（pure forward KL）
* **B**: (\alpha=0.75)
* **C**: (\alpha=0.5)
* **D**: (\alpha=0.0)（pure reverse KL）

固定：
[
m=2,\quad K=8
]

再单独补两个消融：

* span 长度：(m=2) vs. (m=4)
* rollout 长度：(K=8) vs. (K=16)

### 预期观察点

如果方法有效，应看到：

* corrupted-prefix accuracy 上升
* recovery gap 缩小
* rollout 段 teacher / student 分布差异下降
* pure forward KL 或 forward-dominant 混合优于 reverse-dominant 版本

这版实验直接对齐了三件事：
一是训练集采用 **OpenThoughts-114k-math**；二是 teacher clean / student corrupt 的 on-policy 蒸馏形式；三是将 divergence objective 从 JSD 改成了 **forward / reverse KL 的可控混合**。
