# 实验设计：Linear Attention 的 Memory Pollution

## 0. 实验目的

验证以下假设：

> 与 full attention 相比，pure linear attention 更容易受到上下文中**插入的无关 token / span**的影响，其 memory state 更容易发生漂移，进而导致后续 retrieval / QA / 推理性能下降。

更具体地，回答两个问题：

1. 在加入相同插入扰动后，**pure linear 模型**的性能下降是否显著大于 **full attention 模型**？
2. 对 pure linear 模型，这种性能下降是否伴随更明显的 **state memory 漂移**？

---

## 1. 使用的模型

基于论文 *A Systematic Analysis of Hybrid Linear Attention* 中的模型，使用 **340M** 和 **1.3B** 两档。论文系统训练了这两档 pure linear、hybrid 和 transformer 模型；并指出 recall 类任务比普通 LM 指标更能区分 linear 与 full attention。

### 1.1 340M

用于快速验证与扫参：

* Transformer-340M
* Pure Linear-340M：

  * HGRN2
  * GatedDeltaNet
  * GLA 或 RetNet

### 1.2 1.3B

用于主实验与最终结论：

* Transformer-1.3B
* Pure Linear-1.3B：

  * HGRN2
  * GatedDeltaNet
  * GLA 或 RetNet

### 1.3 可选补充

* 少量 hybrid 模型作为桥梁组
  例如：HGRN2-hybrid、GatedDeltaNet-hybrid
  目的是观察 robustness 是否介于 pure linear 与 transformer 之间。论文显示，随着 full-attention 层占比增加，recall 能力会明显上升。

## 2. 使用的数据集

## 2.1 第一阶段：真实 QA / 多选任务

先用你已有的任务：

* ARC-E
* ARC-C
* PIQA
* 可选：OBQA、HellaSwag

### 输入形式


仿照lm-eval-harness的评测方式：

这里**不直接修改问题和选项内容**，而是在题目前或题目中插入无关内容。

---

## 2.2 第二阶段：RULER retrieval 子任务（暂时不考虑进行）

建议补充 **RULER retrieval** 作为更干净的机制验证集。RULER 是一套 long-context synthetic benchmark，包含 retrieval、multi-hop tracing、aggregation 和 QA；其中 retrieval 本质上是 key-value / needle-in-a-haystack 风格任务，非常适合测 memory / recall。

优先使用：

* Single-Key
* Multi-Key
* Multi-Value

原因：

* ground truth 清楚
* 上下文模板化
* 插入无关内容更容易控制
* 更容易把效果解释为 memory pollution，而不是题意被破坏

## 3. 实验变量

## 3.1 扰动方式：插入无关内容


### 方式 A：插入随机 token(首次实验采用这个，测试能跑通)

* 在输入中若干位置插入随机 token
* 作为最简单 baseline

优点：

* 实现最简单

缺点：

* 文本不自然
* 更像 generic noise

### 方式 B：插入无关但流畅的 span

* 从其他样本中抽取与当前问题无关的句子/短段落
* 插入到当前样本输入中

优点：

* 更自然
* 更像真实 rollout 中混入了无关内容
* 更适合作为主实验

### 方式 C：插入相关但无用的 distractor

* 插入与主题表面相关、但对当前答案无帮助的内容

优点：

* 比无关 span 更强
* 更容易造成 state crowding / confusion

---

## 3.2 插入位置

把位置作为一个变量：

* **随机插入**(首次实现)
* **前部插入**：在题目前较早位置插入
* **中部插入**：在题目中间插入
* **问题前紧邻插入**：在 Question 前插入

解释：

* 前部 / 中部插入更适合测试**长期污染是否累积**
* 紧邻问题的插入更像**局部干扰**

---

## 3.3 插入模式

* **单次大块插入**：一次性插入一大段
* **多次小块插入**：把同样总长度拆成多次分散插入

后者更贴近“rollout 过程中持续出现少量错误 token”的场景。

---

## 4. 扰动强度设置

定义插入比例 (r) 为插入 token 数占原始输入长度的比例。

建议：

$$
r \in {0,\ 0.05,\ 0.1,\ 0.2,\ 0.4}
$$

对应：

* 0：clean
* 5%：轻度扰动
* 10%：中低扰动
* 20%：中度扰动
* 40%：重度扰动

实现时：

* 固定原样本
* 额外插入 $k = \lfloor r \cdot L \rfloor$ 个 token 或等长 span
* 在实际eval时，每个样本最好preturb后是一样的

### 控制实验

为区分“长度变长”与“污染”：

* Clean
* 插入占位 token / 重复 token
* 插入无关流畅 span

如果只是长度增加导致掉分，那么后两者应接近；
如果无关内容本身在污染 state，则“无关流畅 span”应更伤。

---

## 5. 计算指标

## 5.1 主指标

### (1) Accuracy

分别计算 clean 和 perturb 后的准确率：

$$
Acc_{\text{clean}}, \quad Acc_{\text{perturb}}
$$

### (2) Accuracy Drop

定义性能下降：

$$
\Delta Acc = Acc_{\text{clean}} - Acc_{\text{perturb}}
$$

重点比较：

* pure linear 的 (\Delta Acc)
* transformer 的 (\Delta Acc)

若假设成立，应有：

$$
\Delta Acc_{\text{linear}} > \Delta Acc_{\text{full}}
$$

---

## 5.2 更稳健的指标

### (3) Shared-clean-correct 子集上的 Accuracy Drop

只在 clean 条件下 **linear 与 transformer 都答对** 的样本子集上比较 perturb 后掉分。

作用：

* 避免 linear baseline 本来较低带来的 floor effect

### (4) Logit Margin Drop

对于单选题，定义：

$$
\text{margin} = \log P(\text{gold}) - \max_{w \neq gold}\log P(w)
$$

比较 clean 与 perturb 后 margin 的变化。

优点：

* 比 accuracy 更细
* 能看到模型是否被逐渐带偏

---

## 6. State Memory 漂移评估

这是验证“memory 被污染”的核心机制指标。

## 6.1 取 state 的位置

对 pure linear 模型，在**读完整个插入后输入、进入最终答题段之前**，提取各层 linear memory state。

记：

* clean 输入后的 state：$\mathbf{S}_{\text{clean}}$
* 插入扰动后的 state：$\mathbf{S}_{\text{perturb}}$

---

## 6.2 漂移指标

第一版采用归一化 L2 距离：

$$
D = \frac{|\mathbf{S}_{\text{perturb}}-\mathbf{S}_{\text{clean}}|^2}{|\mathbf{S}_{\text{clean}}|^2}
$$

若 state 为矩阵，可 flatten 后计算；
也可补充 cosine distance。

---

## 6.3 分层分析

对每一层分别计算：

$$
D^{(\ell)} = \frac{|\mathbf{S}^{(\ell)}_{\text{perturb}}-\mathbf{S}^{(\ell)}_{\text{clean}}|^2}{|\mathbf{S}^{(\ell)}_{\text{clean}}|^2}
$$

分析：

* 哪些层更敏感
* 漂移是否随层数累积

---

## 6.4 漂移与性能的关系

对每个样本记录：

* state drift $D$
* 是否答错
* margin drop

然后分析：

* $D$ 与错误率的关系
* $D$ 与 margin drop 的相关性

若假设成立，应观察到：

* 插入强度越大，state drift 越大
* state drift 越大，越容易出错

---

## 7. 实验流程

### 阶段一：真实 benchmark

* 数据：ARC-E / ARC-C / PIQA
* 扰动：插入随机 token、插入无关流畅 span
* 比较：340M 与 1.3B 的 transformer / pure linear
* 输出：accuracy drop + state drift

### 阶段二：RULER retrieval

* 数据：Single-Key / Multi-Key / Multi-Value
* 扰动：插入无关 token / span
* 重点：观察 retrieval accuracy 与 state drift 的关系

### 阶段三：增强版

* 插入相关 distractor
* 多次小插入 vs 单次大插入
* 可加入 hybrid 模型

---

## 8. 预期结果

若假设成立，预期看到：

1. 在相同插入扰动下，**pure linear 的 accuracy drop 大于 transformer**
2. 该现象在 **1.3B** 上仍成立
3. pure linear 的 **state drift** 显著大于 transformer 对应的表征变化
4. 插入越多、越分散，linear 越脆弱
5. 在 RULER retrieval 上，这种差异会更明显，因为该 benchmark 本身就是为了测 recall / retrieval 能力而设计的。

---

## 9. 本设计的优点

相较于“直接 corrupt 原题 token”，本方案有三个优势：

1. **不直接破坏题意**
2. 更容易把性能下降解释为 **memory pollution / state crowding**
3. 更适合和 RULER retrieval 这类 recall benchmark 结合，机制解释更清楚
