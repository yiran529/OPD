## 1. Motivation

Linear attention 的内部状态是递归累积的。若前缀中出现错误 token，这些错误会被写入 memory / state，并持续影响后续计算。于是误差不仅体现在当前位置，还会通过 state 传播到后续位置，削弱模型的自我纠正能力。

因此，希望在 midtraining 阶段显式训练这样一种能力：即使前缀已经被局部错误污染，模型后续预测仍尽量接近“前缀正确时”的行为，同时内部 state 也尽量不被这些局部错误带偏。

这里不再直接使用模型完整 rollout 得到的 $y_{\mathrm{rollout}}$ 作为错误前缀。原因是：完整 rollout 前缀可能已经发生明显语义漂移，甚至与 ground-truth continuation 不再处于同一语义轨道；若再拿它去和 clean path 做 token-level 对齐，容易引入不可满足的监督目标。

因此，这里改为从 ground-truth prefix $y$ 出发，构造一个**局部扰动后的 corrupted prefix** $\tilde y$。扰动只发生在少数位置上，并且：

1. 优先选择模型最不确定、即熵最高的几个位置；
2. 被选中的位置，token 直接替换为模型在该位置的实际预测 token；
3. 扰动后的前缀仍保持与原始 $y$ 高度接近，从而尽量维持语义连续性与分布自然性。

在此基础上，student 从 $x,\tilde y$ 出发 rollout 出后续轨迹 $\hat z$。teacher 则**不再单独 rollout 一条新轨迹**，而是在每个位置 $t$ 直接使用 **同一个 student 前缀 $\hat z_{<t}$** 作为后续条件，只比较 clean prefix $y$ 与 corrupted prefix $\tilde y$ 在相同后续条件下诱导出的输出分布与内部状态差异。

目标可概括为：

1. 提升 linear attention 在局部前缀污染下的 self-correction 能力；
2. 减少错误 token 对 state 的持续污染；
3. 让训练信号集中刻画“prefix corruption 本身的影响”，而不是混入 teacher / student 两条独立 rollout 轨迹的额外分叉。

## 2. 符号定义

设 $x$ 为输入上下文，$y=(y_1,\dots,y_M)$ 为从 ground-truth continuation 中截取的一段真实前缀片段，长度为 $M$。我们将在 $y$ 上构造一个带局部扰动的 corrupted prefix $\tilde y=(\tilde y_1,\dots,\tilde y_M)$。

### 2.1 基于熵的扰动位置打分

先在 clean prefix 条件下，对每个位置 $i$ 计算模型对下一个 token 的预测分布：

$$
q_i(\cdot) = p_\theta(\cdot \mid x, y_{<i}).
$$

定义该位置的预测熵为

$$
H_i = -\sum_{v \in V} q_i(v) \log q_i(v),
$$

其中 $V$ 为词表。

$H_i$ 越大，说明模型在该位置越不确定，也越可能在真实 rollout 中犯错。因此，我们不再把熵映射为独立采样概率，而是直接把所有位置按熵从高到低排序，并优先挑选熵最大的若干位置进行扰动。

### 2.2 扰动位置的确定

设最终要扰动的位置集合为 $\mathcal I \subseteq \{1,\dots,M\}$。默认做法是：

1. 先计算所有位置的熵分数 $H_1,\dots,H_M$；
2. 按熵从大到小排序；
3. 取前 $K$ 个位置作为扰动位置，其中 $K$ 可以固定，也可以按前缀长度自适应设定。

若采用按比例选取的方式，可写为

$$
K = \min\big(K_{\max},\; \lfloor \rho M \rfloor\big),
$$

其中：

- $\rho$ 为最大扰动比例，例如 $0.1 \sim 0.25$；
- $K_{\max}$ 为绝对数量上限，避免前缀太长时扰动过多。

然后令

$$
\mathcal I = \operatorname{TopK}\big(\{H_i\}_{i=1}^{M}, K\big).
$$

也就是说，只有熵排名前 $K$ 的位置会被扰动，其余位置全部保持 clean token 不变。

这种做法比按概率独立采样更可控：

1. 每个样本的扰动强度稳定，不会因为 Bernoulli 采样而大幅波动；
2. 能确保扰动真正集中在最不确定的位置，而不是“有时命中高熵点，有时又漏掉”；
3. 更方便精确控制总扰动比例。

### 2.3 span 平滑：让扰动更局部、更连续(暂时不实现！)

如果直接取 Top-$K$ 位置，得到的集合 $\mathcal I$ 可能是若干离散散点。为了让 corruption 更接近自然的局部错误，而不是完全零散的单点替换，可以在 Top-$K$ 选择之后再做一次 **span 平滑（span smoothing）**。

具体可用如下规则：

1. 先得到按位置排序后的扰动索引集合 $\mathcal I$；
2. 将距离不超过 $d_{\mathrm{merge}}$ 的相邻位置合并为同一个 span；
3. 对长度为 1 的孤立 span，可选地向左或向右扩展 1 个 token；
4. 对长度超过 $L_{\max}$ 的 span，截断到最大长度 $L_{\max}$；
5. 若扩展后总扰动数超过预算 $K$ 或比例上限 $\rho_{\max}M$，则优先保留平均熵更高的那些 span。

这样，最终真正执行替换的位置集合不一定是最初的精确 Top-$K$ 散点，而是由这些高熵位置出发、经过局部合并和轻微扩展后形成的若干短 span。它的目的不是增加噪声强度，而是让 corruption 更像真实生成错误中的“局部短语偏移”。

### 2.4 扰动 token 的替换方式

在每个位置 $i$，定义模型的实际预测 token 为

$$
\hat y_i^{\mathrm{pred}} = \arg\max_{v \in V} q_i(v).
$$

默认实现中，若 $i \in \mathcal I$，则直接替换为模型实际预测的 token：

$$
\tilde y_i = \hat y_i^{\mathrm{pred}}.
$$

若 $i \notin \mathcal I$，则

$$
\tilde y_i = y_i.
$$

于是可统一写为

$$
\tilde y_i =
\begin{cases}
\hat y_i^{\mathrm{pred}}, & i \in \mathcal I, \\
 y_i, & i \notin \mathcal I.
\end{cases}
$$

这里有两个实现细节需要明确：

1. **若 $\hat y_i^{\mathrm{pred}} = y_i$**，那么该位置虽然被高熵选中，但替换后表面上不发生变化。这是允许的，因为它表示该位置虽不确定，但模型最终 top-1 预测仍与 ground truth 一致。
2. 若希望“被选中的位置一定出现可见 corruption”，可以定义一个实验变体：当 top-1 恰好等于 $y_i$ 时，改用最高概率的非 $y_i$ token 替换。但这不再是本文默认设定。

### 2.5 Student rollout 与 Teacher 条件分布

从 corrupted prefix 出发，student rollout 出一段后续序列

$$
\hat z = (\hat z_1, \dots, \hat z_T), \qquad \hat z \sim p_\theta(\cdot \mid x, \tilde y).
$$

在后续位置 $t$，student 的条件分布为

$$
p^{\mathrm{stu}}_t(\cdot)
= p_\theta(\cdot \mid x, \tilde y, \hat z_{<t}).
$$

teacher **不再自己 rollout 出一条 $z$**。而是在同一个 $\hat z_{<t}$ 条件下，只把前缀中的 corrupted part 换回 clean prefix：

$$
p^{\mathrm{tea}}_t(\cdot)
= p_\theta(\cdot \mid x, y, \hat z_{<t}).
$$

因此，teacher 与 student 在位置 $t$ 的唯一区别是：

- student 看到的是 $x,\tilde y,\hat z_{<t}$；
- teacher 看到的是 $x,y,\hat z_{<t}$。

这样可以把训练目标明确解释为：**在给定同一条后续 rollout 前缀 $\hat z_{<t}$ 时，要求模型对 corrupted prefix 的行为尽量恢复到 clean prefix 下的行为。**

### 2.6 State 的定义

记 linear attention 的内部状态为

$$
\mathrm{state}^{\mathrm{stu}}_t = \mathrm{state}_t(x, \tilde y, \hat z_{<t}),
$$

$$
\mathrm{state}^{\mathrm{tea}}_t = \mathrm{state}_t(x, y, \hat z_{<t}).
$$

若只关注累计 memory / state tensor，则记为

$$
S^{\mathrm{stu}}_t = S_t(x, \tilde y, \hat z_{<t}),
$$

$$
S^{\mathrm{tea}}_t = S_t(x, y, \hat z_{<t}).
$$

## 3. 训练流程

### Step 1: 从 clean prefix 出发计算熵与预测分布

给定输入上下文 $x$ 与 ground-truth prefix $y$，在 teacher-forcing 条件下逐位置计算

$$
q_i(\cdot)=p_\theta(\cdot \mid x,y_{<i}), \qquad i=1,\dots,M.
$$

从而得到每个位置的：

1. 熵 $H_i$；
2. 模型实际预测 token $\hat y_i^{\mathrm{pred}}$。

### Step 2: 直接选取熵最大的若干位置并构造 corrupted prefix

对所有位置按熵从高到低排序，取前 $K$ 个位置组成扰动集合 $\mathcal I$。必要时，再对这些位置做一次 span 平滑，得到最终替换位置。

然后按

$$
\tilde y_i =
\begin{cases}
\hat y_i^{\mathrm{pred}}, & i \in \mathcal I, \\
 y_i, & i \notin \mathcal I
\end{cases}
$$

构造出 $\tilde y$。

### Step 3: 从 corrupted prefix rollout student 轨迹

以 $x,\tilde y$ 为条件，由模型 rollout 出长度为 $T$ 的 student 后续轨迹：

$$
\hat z \sim p_\theta(\cdot \mid x, \tilde y).
$$

这一步仍然保留 on-policy 特征，因为后续轨迹来自模型在 corrupted prefix 条件下的真实 rollout。

### Step 4: 构造 teacher 分布，但不单独 rollout

对于每个后续位置 $t$，teacher 不再生成一条独立轨迹，而是直接在 **同一个** $\hat z_{<t}$ 前缀条件下前向计算：

$$
p^{\mathrm{tea}}_t(\cdot)=p_\theta(\cdot \mid x,y,\hat z_{<t}).
$$

也就是说，teacher 只提供“若前缀仍然是 clean 的，那么在当前这条 student rollout 历史下，模型应该如何预测”的参考分布。

### Step 5: 对齐输出分布与内部状态

在每个位置 $t$，对齐：

1. 输出分布
   $$p^{\mathrm{stu}}_t(\cdot)=p_\theta(\cdot \mid x,\tilde y,\hat z_{<t})$$
   与
   $$p^{\mathrm{tea}}_t(\cdot)=p_\theta(\cdot \mid x,y,\hat z_{<t});$$
2. 内部状态
   $$S_t(x,\tilde y,\hat z_{<t})$$
   与
   $$S_t(x,y,\hat z_{<t}).$$

这样，训练目标不再混入 teacher / student 两条不同 rollout 路径的差异，而是聚焦于“clean prefix 与 corrupted prefix 在相同后续条件下造成的偏移”。

## 4. 扰动实现细节

这一节专门说明扰动部分如何实现，避免在训练时把“扰动”写成一个过于抽象的黑箱步骤。

### 4.1 为什么直接选熵最高的位置

若某个位置的预测熵高，说明模型在该位置本来就不确定，也更容易在真实 rollout 中犯错。因此，直接选熵最高的若干位置，相当于显式挑出模型最脆弱、最容易出错的局部点来构造 corruption。

这样做相对于按概率采样有几个优势：

1. **扰动预算固定**：每个样本大致会扰动同样数量的 token；
2. **目标更稳定**：不会因为采样噪声导致样本间 corruption 强度差异过大；
3. **更聚焦 hardest positions**：训练始终集中在模型最不确定的位置，而不是平均分配到一些并不关键的点上。

换言之，这里的 corruption 不是随机噪声，而是**由模型自身不确定性排序后挑选出的、最需要学习恢复的局部错误**。

### 4.2 一个可直接实现的默认算法

设 prefix 长度为 $M$，词表大小为 $|V|$。默认实现如下：

1. 对 $i=1,\dots,M$：
   1. 计算 $q_i(\cdot)=p_\theta(\cdot\mid x,y_{<i})$；
   2. 计算熵 $H_i$；
   3. 取 $\hat y_i^{\mathrm{pred}}=\arg\max q_i$。
2. 设定扰动预算
   $$
   K = \min(K_{\max},\lfloor \rho M \rfloor).
   $$
3. 取熵最大的前 $K$ 个位置，得到初始扰动集合 $\mathcal I_{\mathrm{raw}}$。
4. 【注意这一步骤暂时不需要实现】若需要更连续的局部扰动，则对 $\mathcal I_{\mathrm{raw}}$ 做一次 span 平滑：
   1. 将距离不超过 $d_{\mathrm{merge}}$ 的位置合并成 span；
   2. 对长度为 1 的 span，可向左或向右扩展 1 个 token；
   3. 限制每个 span 的长度不超过 $L_{\max}$；
   4. 若总替换数超过预算 $K$ 或比例上限 $\rho_{\max}M$，则删去平均熵较低的 span，得到最终扰动集合 $\mathcal I$。
5. 最终按 $i \in \mathcal I$ 将被选中位置替换为 $\hat y_i^{\mathrm{pred}}$，得到 $\tilde y$。

一个实践上容易跑通的超参数组合是：

- $\rho = 0.25$
- $d_{\mathrm{merge}} = 1$
- $L_{\max}=4$
- $\rho_{\max}=0.25$

即：优先扰动前缀中大约前 25% 的最高熵位置；合并相邻高熵点后，单个连续扰动 span 不超过 4 个 token；整段 prefix 的最终总扰动比例不超过 25%。

### 4.3 为什么替换为模型自己的预测 token

替换为模型自己的预测 token，而不是随机 token 或人工规则替换，有三个好处：

1. **自然性更强**：被替换进去的是模型在该 clean 局部上下文下本来就最可能犯出的错误；
2. **分布更贴近真实推理错误**：这种错误更接近测试时模型实际会写出的内容；
3. **恢复目标更合理**：模型需要学会从“自己会犯的局部错误”中恢复，而不是从完全异质的随机噪声中恢复。

### 4.4 是否使用 argmax 还是采样

本文默认用

$$
\hat y_i^{\mathrm{pred}} = \arg\max q_i
$$

作为“模型实际预测 token”，原因是它定义清晰、方差低、实现稳定。

若希望让 corruption 更接近真实 sampling rollout，也可以把替换 token 改为从 $q_i$ 中做 top-$p$ 或 top-$k$ 采样得到。但这会显著增加扰动方差。除非实验已经表明 greedy corruption 太保守，否则建议默认先用 argmax 版。

### 4.5 如何避免扰动过强

为了防止 corrupted prefix 偏离 clean prefix 太远，建议同时加上以下约束：

1. **总扰动比例上限**：例如整段 prefix 最多替换 15% 到 25% 的 token；
2. **单个 span 长度上限**：例如不超过 3 到 4 个 token；
3. **前缀前部保护**：可选地对最开头若干 token 不做扰动，避免整个语境过早漂移；
4. **高熵但预测正确的位置允许不变**：若被选中位置的模型 top-1 恰好就是 ground truth，则允许该位置保持不变，不强行引入非自然错误。

这些约束的目标是：corruption 要足够强，能让模型看到需要“恢复”的局部错误；但又不能强到把原句整体打碎，导致监督目标重新变得不自然。

## 5. Loss 设计

总目标由两部分组成：

1. 基于输出分布的 JSD 对齐项；
2. 基于内部状态的 state 对齐项。

### 5.1 主 loss：Jensen-Shannon Divergence

在每个后续位置 $t$，定义

$$
p_t = p^{\mathrm{stu}}_t(\cdot)=p_\theta(\cdot \mid x,\tilde y,\hat z_{<t}),
$$

$$
q_t = sg\big(p^{\mathrm{tea}}_t(\cdot)\big)=sg\big(p_\theta(\cdot \mid x,y,\hat z_{<t})\big),
$$

其中 $sg(\cdot)$ 表示对 teacher 分布 stop-gradient。

令

$$
m_t = \frac{1}{2}(p_t + q_t).
$$

则逐位置的 JSD 定义为

$$
\mathrm{JSD}(p_t \| q_t)
= \frac{1}{2}\mathrm{KL}(p_t \| m_t) + \frac{1}{2}\mathrm{KL}(q_t \| m_t).
$$

因此主 loss 为

$$
\mathcal{L}_{\mathrm{JSD}} = \sum_{t=1}^{T} \mathrm{JSD}(p_t \| q_t).
$$

采用 JSD 而不是单向 KL 的原因是：

1. 它是对称的，不会像单向 KL 那样明显偏向某一侧；
2. 当 teacher 与 student 分布差异较大时，JSD 往往更稳定；
3. 它更符合这里的目标：我们不是把 teacher 当作绝对正确的外部模型，而是在同一模型参数下，用 clean-prefix 条件分布去约束 corrupted-prefix 条件分布。

JSD loss也可以有时间步相关权重。

### 5.2 State 对齐 loss

除输出分布外，再加入一个针对内部状态的对齐项。

对后续位置 $t$，设

- $a_t = S_t(x,\tilde y,\hat z_{<t})$ 为 student 路径状态；
- $b_t = sg\big(S_t(x,y,\hat z_{<t})\big)$ 为 teacher 路径状态，其中 teacher state 使用 stop-gradient。

定义逐步权重为

$$
w_t = \left(\frac{t+1}{T}\right).
$$

同时定义

$$
\text{cos\_loss}_t = 1 - \mathrm{cosine\_similarity}(a_t,b_t),
$$

$$
\text{norm\_loss}_t = \left(\lVert a_t \rVert - \lVert b_t \rVert\right)^2.
$$

则每个位置 $t$ 的 state 对齐 loss 为

$$
\mathcal{L}_{\mathrm{state},t}
= w_t \cdot \left(\text{cos\_loss}_t + 0.1 \cdot \text{norm\_loss}_t\right).
$$

对所有 layer 的所有 state tensor 取平均，并在时间维度上求和，得到

$$
\mathcal{L}_{\mathrm{state}} = \sum_{t=1}^{T} \mathcal{L}_{\mathrm{state},t}.
$$

该项的作用是：不仅让 logits 接近，还要求 corrupted prefix 诱导出的内部 memory 在相同 rollout 历史下逐步恢复到 clean prefix 对应的 memory 方向与尺度，从而减轻 state contamination。

---

## 6. 总目标

最终训练目标为

$$
\mathcal{L} = \mathcal{L}_{\mathrm{JSD}} + \lambda_{\mathrm{state}} \mathcal{L}_{\mathrm{state}}.
$$

其中 $\lambda_{\mathrm{state}}$ 控制状态对齐项的强度。

若训练初期发现 state 对齐过强、影响语言建模稳定性，可采用 warmup：

$$
\lambda_{\mathrm{state}}(s)
= \lambda_{\mathrm{state}}^{\max} \cdot \min\left(1, \frac{s}{S_{\mathrm{warm}}}\right),
$$

其中 $s$ 为当前训练步，$S_{\mathrm{warm}}$ 为 warmup 步数。

---

## 7. 总结

整个方案的核心不再是使用完整 $y_{\mathrm{rollout}}$ 与 clean path 做双轨 rollout 对齐，而是：

1. 从 ground-truth prefix $y$ 出发；
2. 用**熵排序后的 Top-$K$ 局部扰动**构造 corrupted prefix $\tilde y$；
3. 被选中位置的 token 直接替换为模型在该位置的实际预测 token；
4. 从 $x,\tilde y$ 出发 rollout 得到 student 后续轨迹 $\hat z$；
5. teacher **不再单独 rollout**，而是在每个位置 $t$ 直接使用
   $$p_\theta(\cdot \mid x,y,\hat z_{<t})$$
   作为 clean reference distribution；
6. 用 JSD 对齐输出分布，并用 state loss 对齐内部 memory / state。

这样做有三个直接好处：

1. **前缀更自然**：corruption 只发生在少量最高熵位置，且替换为模型自己的预测 token；
2. **监督更干净**：teacher 与 student 共享同一个 $\hat z_{<t}$，避免把两条独立 rollout 轨迹的分叉误当成训练目标；
3. **目标更贴合“恢复”**：模型真正学习的是：在后续 rollout 已经沿着当前历史展开的情况下，如何把 corrupted prefix 诱导出的行为与 state 拉回到 clean prefix 下的行为与 state。
