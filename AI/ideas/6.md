## 实验设计

### 模型

采用 **Qwen/Qwen3.5-0.8B** 作为底座模型。该模型是 24 层 hybrid 架构，适合做小规模 reasoning finetuning 与方法验证。初版仍在 `LinearOPSD` 路径中实现，不额外新开训练框架，也不新增新的 `conditioning_mode`；而是直接改现有 `linear_opsd` 这条训练路径。

### 训练数据

主训练集仍采用 **OpenThoughts-114k-math**。每条样本抽取：

\[
x = \text{problem}, \qquad y = \text{solution}
\]

其中 \( y \) 是完整 gold CoT。初版仍聚焦数学推理，不额外拆 answer-only 监督，也不引入额外 CE anchor，主目标仍是 rollout 段上的分布蒸馏。

备选小规模 sanity-check 数据集仍可保留：

- **GSM8K**
- **Bespoke-Stratos-17k**

但不作为主实验。

### 核心思路

这一版方法的目标，不再是手工选择若干 token 位置并显式替换错误 token；而是让 student 在一段正确的 gold prefix 之后，先用一个更 careless 的解码策略短暂 rollout，**自然地产生一个局部错误前缀**，然后立刻切回正常解码策略继续 rollout。初版中，切回正常 decoding 后的这段 recovery rollout 本身就是蒸馏目标段，因此 recovery rollout 的长度直接等于 KD 计算长度。

整个训练流程可以直接概括为 3 步：

1. 先截取一段正确的 gold prefix，作为 rollout 起点之前的 clean context。
2. student 从这个 gold prefix 出发，先用 careless decoding（例如更高 temperature、top-p/top-k sampling）短 rollout 一小段，长度记为 \(m\)。这一小段的目标不是直接监督，而是把 student 自然带到一个可能偏离 gold 的污染状态。
3. 然后 student 立刻切回正常 decoding（初版优先用 greedy）继续 rollout；teacher 不单独 rollout，而是在看到 `problem + gt solution + student 当前 reasoning trace + 同一段 recovery rollout` 的前提下，对这整段 recovery rollout 提供 next-token supervision。

因此，训练中的两条条件上下文分别是：

- **student**：`problem + gold prefix + careless rollout prefix`
- **teacher**：`problem + gt solution + student current reasoning trace + student rollout`

其中 teacher 的 prompt 要明确说明：

- 已提供正确参考解答；
- 后面会提供 student 当前的推理轨迹；
- student 当前轨迹中的局部错误状态来自 student 自己的一小段 careless rollout，而不是人工替换 token；
- teacher 需要基于完整 gold reasoning 去判断 student 的轨迹偏差，并给出更合理的 next-token 分布。

teacher **不单独 rollout**。  
训练仍然沿用 OPSD / current `LinearOPSD` 的 shared-rollout 形式：student 先自己走出一小段 careless prefix，再切回正常 decoding 继续 rollout；teacher 再在看到同一段 student rollout 的前提下做 teacher forward，并只在正常 decoding 恢复后的短窗口内对 student 做蒸馏。

### teacher / student

- **student**：当前训练中的模型，参数记为 \( \theta_S \)
- **teacher**：teacher 策略可配置，参数记为 \( \theta_T \)

teacher 仍保留三种可切换策略：

- **dynamic teacher**：teacher 与当前 student 共享同一套实时参数
- **fixed teacher**：teacher 固定为初始策略
- **EMA teacher**：teacher 为 student 的指数滑动平均

默认仍优先跑 **dynamic teacher**，同时保留 `fixed teacher` 与 `EMA teacher` 作为实验开关。

在 rollout 位置 \( t \)，student 和 teacher 的条件分布写为：

\[
p_t = p_{\theta_S}(\cdot \mid x, \tilde y^S_{<t})
\]

\[
q_t = p_{\theta_T}(\cdot \mid x, y, \bar y_{<\tau}, \hat y_{\tau:t-1})
\]

其中：

- \( \tilde y^S_{<\tau} \) 是由 `gold prefix + careless rollout` 构成的 student polluted prefix
- \( \hat y_{\tau:t-1} \) 是 student 在该污染状态下、切回正常 decoding 后真实 rollout 出来的历史
- teacher 额外拿到完整 gold solution \( y \)

这意味着 teacher 的 privileged information 不再只是“某些被手工替换位置的 clean token”，而是“完整正确解答 + student 自然生成出来的错误轨迹”。

### prompt 设计

这一版 `linear_opsd` 中，teacher prompt 的自然语言构造方式尽量参考原始 `opsd` 路径，而不是纯 token patch 形式。  
特别是 teacher prompt 的前半段，尽量保留原 `opsd` 里已经存在的重叠结构：

- `Problem: ...`
- `Here is a reference solution to this problem:`
- `=== Reference Solution Begin ===`
- `=== Reference Solution End ===`
- `After reading the reference solution above ...`

teacher 输入建议组织为：

1. `problem`
2. `gt solution`
3. `student 当前 reasoning trace`
4. `student rollout`

这一版不再需要 `<corrupt>` 这类人工标记，但 teacher trace 中会显式加入两个边界标记：

```text
<careless>
<recovery>
```

teacher 看到的 student trace 形态应为：

```text
[gold prefix text] <careless> [careless prefix text] <recovery> [recovery rollout text]
```

teacher 的 instruction 需要显式写明：

- 参考解答是正确的；
- 后面给出的 student trace 中，`<careless>` 之后是 student 自己生成的 careless prefix；
- `<recovery>` 之后是 student 切回正常 decoding 后的 recovery rollout；
- 这段 careless rollout 的作用是制造更真实的 exposure bias / polluted state；
- teacher 应基于参考解答理解 student 当前已经偏离到什么状态，并对 student 当前 rollout 位置给出更合理的 token 分布。

核心要求是：teacher 的输出空间仍然是“同一个 continuation 任务”的 next-token 分布，而不是改成另一个从头回答问题的任务。

更具体地说，teacher 不是重新“从零解题”，而是在更强上下文下，继续对 student 当前这条 reasoning trajectory 的后续 token 进行判断。

### corruption 与 rollout 方案

这一版不再使用 `5.md` 中“从同一样本其他位置随机抽 token span 替换”的 corruption 方案，也不再使用“基于熵筛高熵位置后手工替换错误 token”的方案。  
新的目标是构造**更像 student 自然生成失误的局部错误前缀**。

只污染 `solution`，不污染 `problem`。

#### 1. 截取一段正确的 gold prefix

对每条样本，从 `solution` 中截取一段前缀，记为：

\[
y_{<\tau}
\]

这里 \( \tau \) 是 careless rollout 开始前的位置。  
初版中，\( \tau \) 不需要由熵驱动决定，而是直接按 solution 内的某个位置截断，例如按长度比例在中前段取一个位置。

这一步的目标很简单：保证 student 在进入 careless rollout 之前，所处状态仍然是 clean 的、teacher-forced 的 gold state。

#### 2. careless rollout 生成局部错误前缀

student 从 `problem + y_{<\tau}` 出发，用一个更 careless 的解码策略短 rollout 一小段，长度为 \(m\)：

\[
\hat y^{\text{careless}}_{\tau:\tau+m-1}
\]

这里的 careless decoding 可以用：

- 更高的 temperature
- top-p sampling
- top-k sampling
- 或它们的组合

这一步的目标不是获得高质量 continuation，而是**有意提高 student 在局部进入错误状态的概率**。

careless 段之后，student 的前缀变成：

\[
\tilde y_{<\tau+m}=
[y_{<\tau}, \hat y^{\text{careless}}_{\tau:\tau+m-1}]
\]

这就是后续 rollout 的 polluted prefix。

#### 3. 偏离判定

如果 careless rollout 恰好没有偏离 gold，即：

\[
\hat y^{\text{careless}}_{\tau:\tau+m-1}=
y_{\tau:\tau+m-1}
\]

那么这条 careless 段没有真正制造错误状态。  
初版建议采用宽松但直接的处理：

- 重采若干次；
- 若仍然没有偏离 gold，则该样本该步跳过 OPD supervision，或退化为不贡献 KD loss。

这里的核心原则是：**我们关心的是 student 自己自然走进错误状态后的恢复能力**，而不是为了强行训练而保留并未偏离 gold 的样本。

#### 4. 切回正常 decoding

一旦 careless 段结束，student 立刻切回正常 decoding 策略。  
初版优先用：

- **greedy decoding**

此时 student 从 polluted prefix

\[
\tilde y_{<\tau+m}
\]

继续 rollout，生成长度为 \(K\) 的后续 token：

\[
\hat y_{\tau+m:\tau+m+K-1}
\]

这里的语义是：careless 段只负责把 state 带偏；真正要训练 student 恢复能力的，是后面的正常 rollout 段。

#### 5. recovery rollout 直接作为 KD 段

初版不再单独引入“更长 rollout + 额外 loss mask”的机制。  
更直接的做法是把切回正常 decoding 后的 recovery rollout 长度直接定义为 \(K\)，并只对这 \(K\) 个 token 做蒸馏：

\[
t \in [\tau+m, \tau+m+K-1]
\]

也就是说：

- careless 段只负责把 state 带偏；
- recovery 段只负责恢复；
- KD 的计算长度就等于 recovery rollout 的长度。

这样实现更简单，也更符合方法直觉：我们关心的就是“从 polluted state 出发，接下来这小段正常 rollout 的恢复质量”。

### 训练时 teacher 的条件构造

在正常 decoding 阶段的任意位置 \( t \in [\tau+m, \tau+m+K-1] \)，student 实际看到：

\[
\tilde y^S_{<t} = [y_{<\tau}, \hat y^{\text{careless}}_{\tau:\tau+m-1}, \hat y_{\tau+m:t-1}]
\]

teacher 则看到：

\[
\tilde y^T_{<t} =
\bigl[
x,\;
y,\;
y_{<\tau},\;
\hat y^{\text{careless}}_{\tau:\tau+m-1},\;
\hat y_{\tau+m:t-1}
\bigr]
\]

其中：

- \( y \) 是完整 gold solution
- \( y_{<\tau} \) 是 gold prefix
- \( \hat y^{\text{careless}}_{\tau:\tau+m-1} \) 是 student 自然生成的 careless 局部错误前缀
- student 在切回正常 decoding 后的 rollout 历史直接拼接在 teacher prompt 末尾

因此 teacher 的 conditioning 是“完整正确解答 + student 自然产生的错误状态 + student rollout 历史”，而不是“对同一 prefix 做 token patch”。

这里最重要的约束是：

- student rollout 只有一条，就是 student 自己生成出来的那条轨迹；
- teacher 不生成另一条 teacher rollout；
- teacher 只是在更强的条件上下文下，对这同一条 student rollout 提供蒸馏信号。

### 输入示例

下面给出一个简化示例。设原题与 gold solution 为：

```text
Problem: Solve 2x + 3 = 11.

Gold solution:
2x + 3 = 11, so 2x = 8, hence x = 4.
```

假设先取一段正确的 gold prefix：

```text
Problem: Solve 2x + 3 = 11.

2x + 3 = 11, so
```

然后 student 用 careless decoding 短暂 rollout 两个 token，恰好生成：

```text
 2x = 6
```

于是 student 进入了一个自然形成的局部错误状态。  
此时切回正常 decoding，并且 student 后续继续生成：

```text
 , hence x = 3. Therefore the answer is \boxed{3}.
```

则 student 实际输入示例为：

```text
Problem: Solve 2x + 3 = 11.

Please reason step by step, and put your final answer within \boxed{}.

2x + 3 = 11, so 2x = 6
```

teacher 输入示例为：

```text
Problem: Solve 2x + 3 = 11.

Here is a reference solution to this problem:
=== Reference Solution Begin ===
2x + 3 = 11, so 2x = 8, hence x = 4.
=== Reference Solution End ===

After reading the reference solution above, make sure you truly understand the reasoning behind each step. Do not copy it verbatim. In the student's trace below, the text after <careless> was generated with a more careless decoding strategy from an otherwise correct prefix, and the text after <recovery> is the student's subsequent recovery rollout under normal decoding. Use the reference solution to judge where the student has drifted and provide a better next-token distribution while continuing the same trajectory.

Here is the student's current reasoning trace:
2x + 3 = 11, so <careless> 2x = 6 <recovery> , hence x = 3. Therefore the answer is \boxed{3}.
```

这个 teacher 示例有意与原 `opsd` 路径保持较大重叠：

- 前半段仍然是 `Problem + Reference Solution`
- 中间仍然保留 `After reading ...` 这一类过渡指令
- 只是后半段不再要求 teacher 独立重新推理，而是给出带 `<careless>` / `<recovery>` 边界的 student reasoning trace

这里 teacher 没有自己的独立 rollout；它只是基于更强的上下文，对同一段 student rollout 提供 next-token supervision。

### 损失函数

不使用 CE，只保留蒸馏损失。  
在 student 切回正常 decoding 后的 recovery rollout 段上定义：

\[
L_{\mathrm{KD}}=
\frac{1}{K}
\sum_{t=\tau+m}^{\tau+m+K-1}
\Bigl[
\alpha \, \mathrm{KL}(q_t \,\|\, p_t)
+
(1-\alpha)\, \mathrm{KL}(p_t \,\|\, q_t)
\Bigr]
\]

其中：

\[
\alpha \in [0,1]
\]

是 forward / reverse KL 的混合系数。

总损失即：

\[
L = L_{\mathrm{KD}}
\]

初版主消融轴仍保留：

\[
\alpha \in \{1.0,\ 0.75,\ 0.5,\ 0.25,\ 0.0\}
\]

其中：

- \( \alpha = 1 \)：纯 forward KL
- \( \alpha = 0 \)：纯 reverse KL
- \( \alpha = 0.5 \)：对称混合

初版默认仍先用 **pure forward KL** 作为最重要对照。

### 实现约束

这一版最大的实现变化是：局部错误前缀不再能在 data collator 内静态构造，因为它依赖一次真实 student rollout，并且需要在同一条样本内切换 decoding 策略。

因此在 `LinearOPSD` 的实现上，应遵循：

- 不新增新的 `conditioning_mode`
- 直接改现有 `linear_opsd`
- teacher prompt 组织方式尽量复用 / 参考现有 `opsd` 这条 prompt 风格
- careless rollout 的生成逻辑应放在更靠近 trainer / rollout 的位置，而不是继续写成纯静态 collator 逻辑
- teacher trace 中显式加入 `<careless>` 与 `<recovery>` 作为边界标记
- 若 careless 段没有真正偏离 gold，则允许重采；若仍未偏离，则该样本该步跳过 KD 或退化为不贡献 OPD loss
- 不需要为了兼容旧实现而保留手工 corruption point、replacement token、`<corrupt>` 标记等路径

### 训练细节

- teacher 策略可在 dynamic / fixed / EMA 之间切换
- 每条样本每步先保留一段 gold prefix，再生成一小段 careless rollout
- careless rollout 以更高 temperature / top-p / top-k 采样为主，用来制造自然错误状态
- 若 careless rollout 与 gold 完全一致，则重采或跳过该样本该步监督
- careless 段结束后，student 立刻切回正常 decoding
- teacher trace 显式写成 `gold prefix + <careless> + careless prefix + <recovery> + recovery rollout`
- student 正常 rollout 解码初版优先使用 greedy
- teacher 与 student 共享同一条 student rollout 轨迹
- recovery rollout 的长度 \( K \) 直接等于蒸馏长度
- `LinearOPSD` 路径仍以 full-vocabulary logit distillation 为主

### 评测

#### 1. 主评测：error-correction robustness

在 held-out math 样本上固定相同的 `gold prefix + careless rollout + normal recovery` 协议，报告：

- clean prefix 下的解题准确率
- corrupted prefix 下的解题准确率
- recovery gap

\[
\text{recovery gap} = \text{clean acc} - \text{corrupted acc}
\]

- rollout 段平均

\[
\mathrm{KL}(q_t \,\|\, p_t), \qquad \mathrm{KL}(p_t \,\|\, q_t)
\]

以及混合损失

\[
L_{\mathrm{KD}}
\]

这一组指标仍是主指标，用来衡量模型是否更会从局部错误 CoT 中恢复。

#### 2. benchmark

- **GSM8K**
- **MATH-500**
- **MMLU-Pro**
- **IFEval**

其中前两者看数学泛化，后两者看 general capability / instruction-following 是否被明显破坏。

### 最小实验矩阵

主实验先只跑以下几组：

- **A**：\( \alpha = 1.0 \)，`rollout_decoding=sample`
- **B**：\( \alpha = 0.75 \)，`rollout_decoding=sample`
- **C**：\( \alpha = 0.5 \)，`rollout_decoding=sample`
- **D**：\( \alpha = 1.0 \)，`rollout_decoding=greedy`

然后再做以下小规模消融：

- careless rollout 长度：\( m \in \{4, 8\} \)
- recovery rollout 长度：\( K \in \{8, 16\} \)
- careless decoding 强度：低温 vs 高温 / 更小 top-p
- teacher 策略：dynamic vs EMA vs fixed

### 与 5.md 的主要区别

相较于 `5.md`，这一版的变化是：

- teacher 不再只拿 clean local patch，而是拿完整 gold solution
- teacher prompt 中显式加入 student 的 careless prefix 与 recovery rollout
- 用 `<careless>` 和 `<recovery>` 显式标记 student trace 的阶段边界
- corruption 不再是随机 span 替换或基于熵的手工 token 替换，而是 student 自己的 careless rollout
- 方法目标从“local patch supervision”进一步转向“natural polluted-state recovery”
