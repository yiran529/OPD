## 实验设计

### 模型

采用 **Qwen/Qwen3.5-0.8B** 作为初始底座模型。该模型是 24 层 hybrid 架构，适合做小规模 reasoning finetuning 与方法验证；后续可直接用同一路径扩展到 **Qwen/Qwen3.5-2B**。初版仍在 `LinearOPSD` 路径中实现，不额外新开训练框架，也不新增新的 `conditioning_mode`；而是直接改现有 `linear_opsd` 这条训练路径。

### 训练数据

主训练集仍采用 **OpenThoughts-114k-math**。每条样本抽取：

\[
x = \text{problem}, \qquad y = \text{solution}
\]

其中 \( y \) 是完整 gold CoT。初版仍聚焦数学推理，不额外拆 answer-only 监督，也不引入额外 CE anchor，主目标仍是 rollout 段上的分布蒸馏。

备选小规模 sanity-check 数据集仍可保留：

- **GSM8K**
- **Bespoke-Stratos-17k**

但不作为主实验。

### 核心思路

这一版方法的目标，不再是手工选择若干 token 位置并显式替换错误 token；而是让 student 在一段正确的 gold prefix 之后，可选地用一个更 careless 的解码策略短暂 rollout，**自然地产生一个局部 sampled tail**，然后立刻切回正常解码策略继续 rollout。当前实现同时混入 clean prefix 样本，避免每条训练样本都呈现为显式纠错场景。切回正常 decoding 后的这段 recovery rollout 本身就是蒸馏目标段，因此 recovery rollout 的长度直接等于 KD 计算长度。

整个训练流程可以直接概括为 3 步：

1. 先截取一段正确的 gold prefix，作为 rollout 起点之前的 clean context。
2. 根据 clean/mild/hard mixture 决定是否让 student 从这个 gold prefix 出发，用 careless decoding（例如更高 temperature、top-p/top-k sampling）短 rollout 一小段，长度记为 \(m\)。clean 模式下 \(m=0\)，没有 sampled tail；mild/hard 模式下 sampled tail 的目标不是直接监督，而是把 student 自然带到一个可能偏离 gold 的状态。
3. 然后 student 立刻切回正常 decoding（初版优先用 greedy）继续 rollout；teacher 不单独 rollout，而是在看到 `problem + gt solution + student 当前 reasoning trace + 同一段 recovery rollout` 的前提下，对这整段 recovery rollout 提供 next-token supervision。

因此，训练中的两条条件上下文分别是：

- **student**：`problem + gold prefix + optional sampled tail`
- **teacher**：`problem + gt solution + student current partial solution + student rollout`

其中 teacher 的 prompt 要明确说明：

- 已提供正确参考解答；
- 后面会提供 student 当前的推理轨迹；
- marked sampled tail 如果存在，则来自 student 自己的一小段 careless rollout，而不是人工替换 token；
- teacher 需要把 reference solution 作为 private context，自然继续当前 partial solution 的下一步，而不是重新从头解题或输出显式纠错话术。

teacher **不单独 rollout**。  
训练仍然沿用 OPSD / current `LinearOPSD` 的 shared-rollout 形式：student 先得到一段 gold prefix，并在 mild/hard 模式下自己走出一小段 sampled tail，再切回正常 decoding 继续 rollout；teacher 再在看到同一段 student rollout 的前提下做 teacher forward，并只在正常 decoding 恢复后的短窗口内对 student 做蒸馏。

### teacher / student

- **student**：当前训练中的模型，参数记为 \( \theta_S \)
- **teacher**：teacher 策略可配置，参数记为 \( \theta_T \)

teacher 仍保留三种可切换策略：

- **dynamic teacher**：teacher 与当前 student 共享同一套实时参数
- **fixed teacher**：teacher 固定为初始策略
- **EMA teacher**：teacher 为 student 的指数滑动平均

当前脚本用 `fixed_teacher` 。

在 rollout 位置 \( t \)，student 和 teacher 的条件分布写为：

\[
p_t = p_{\theta_S}(\cdot \mid x, \tilde y^S_{<t})
\]

\[
q_t = p_{\theta_T}(\cdot \mid x, y, \bar y_{<\tau}, \hat y_{\tau:t-1})
\]

其中：

- \( \tilde y^S_{<\tau} \) 是由 `gold prefix + optional sampled tail` 构成的 student prefix
- \( \hat y_{\tau:t-1} \) 是 student 在当前 prefix 状态下、切回正常 decoding 后真实 rollout 出来的历史
- teacher 额外拿到完整 gold solution \( y \)

这意味着 teacher 的 privileged information 不再只是“某些被手工替换位置的 clean token”，而是“完整正确解答 + student 当前真实 partial trajectory”。

### prompt 设计

这一版 `linear_opsd` 中，teacher prompt 的自然语言构造方式尽量参考原始 `opsd` 路径，而不是纯 token patch 形式。  
特别是 teacher prompt 的前半段，尽量保留原 `opsd` 里已经存在的重叠结构：

- `Problem: ...`
- `Here is a reference solution to this problem:`
- `=== Reference Solution Begin ===`
- `=== Reference Solution End ===`
- continuation-oriented instruction

teacher 输入建议组织为：

1. `problem`
2. `gt solution`
3. `student 当前 reasoning trace`
4. `student rollout`

这一版不再需要 `<corrupt>` 这类人工标记。当前实现也不再默认使用 XML/HTML-like 的 `<careless>` / `<recovery>` 边界，因为这类 marker 容易扰乱 continuation 风格，并诱导模型学到显式纠错话术。teacher trace 只在存在 sampled tail 时加入一个低干扰 marker，默认是：

```text
[recent sampled tail begins here]
```

teacher 看到的 student trace 形态应为：

```text
[gold prefix text] [recent sampled tail begins here] [sampled tail text] [student recovery rollout text]
```

如果是 clean 模式，则没有 marker，也没有 sampled tail：

```text
[gold prefix text] [student recovery rollout text]
```

teacher 的 instruction 需要显式写明：

- 参考解答是正确的，但只作为 private context；
- 后面给出的 partial solution 是要继续的当前轨迹；
- 如果出现 sampled-tail marker，则 marker 后的短 tail 可能包含局部数学或措辞不一致；
- teacher 应直接从 partial solution 停止的位置继续下一步数学推理；
- 如果 recent tail 不一致，teacher 应通过后续 continuation 恢复数学连贯性，而不是输出“我来纠正”“let's check”等显式纠错话术。

核心要求是：teacher 的输出空间仍然是“同一个 continuation 任务”的 next-token 分布，而不是改成另一个从头回答问题的任务。

更具体地说，teacher 不是重新“从零解题”，也不是写一段纠错说明，而是在更强上下文下，继续对 student 当前这条 reasoning trajectory 的后续 token 进行判断。

### corruption 与 rollout 方案

这一版不再使用 `5.md` 中“从同一样本其他位置随机抽 token span 替换”的 corruption 方案，也不再使用“基于熵筛高熵位置后手工替换错误 token”的方案。  
新的目标是构造**更像 student 自然生成失误的局部 sampled tail / prefix state**。

只污染 `solution`，不污染 `problem`。

#### 1. 截取一段正确的 gold prefix

对每条样本，从 `solution` 中截取一段前缀，记为：

\[
y_{<\tau}
\]

这里 \( \tau \) 是 optional sampled tail 开始前的位置。
初版中，\( \tau \) 不需要由熵驱动决定，而是直接按 solution 内的某个位置截断，例如按长度比例在中前段取一个位置。

这一步的目标很简单：保证 student 在进入 optional sampled tail 之前，所处状态仍然是 clean 的、teacher-forced 的 gold state。

#### 2. clean / mild / hard prefix mixture

当前实现不是每条样本都强制生成 sampled tail，而是按比例采样三种 prefix 模式：

- **clean**：只使用 gold prefix，不加入 sampled tail；
- **mild**：加入较短 sampled tail，长度由 `linear_opsd_mild_careless_rollout_len` 控制；
- **hard**：加入较长 sampled tail，长度由 `careless_rollout_len` 控制。

hard 比例不单独配置，而是：

\[
p_{\mathrm{hard}} = 1 - p_{\mathrm{clean}} - p_{\mathrm{mild}}
\]

这样做的目的，是让训练分布同时包含正常 continuation、轻微偏移和明显偏移，避免模型每次都看到“需要纠正”的场景。

#### 3. sampled-tail rollout

在 mild/hard 模式下，student 从 `problem + y_{<\tau}` 出发，用一个更 careless 的解码策略短 rollout 一小段，长度为 \(m\)：

\[
\hat y^{\text{careless}}_{\tau:\tau+m-1}
\]

这里的 careless decoding 可以用：

- 更高的 temperature
- top-p sampling
- top-k sampling
- 或它们的组合

这一步的目标不是获得高质量 continuation，而是**有意提高 student 在局部进入偏移状态的概率**。

sampled tail 之后，student 的前缀变成：

\[
\tilde y_{<\tau+m}=
[y_{<\tau}, \hat y^{\text{careless}}_{\tau:\tau+m-1}]
\]

这就是后续 rollout 的 sampled-tail prefix。它可能是 polluted prefix，也可能只是轻微风格偏移。

#### 4. 偏离判定

如果 mild/hard 模式下的 sampled tail 恰好没有偏离 gold，即：

\[
\hat y^{\text{careless}}_{\tau:\tau+m-1}=
y_{\tau:\tau+m-1}
\]

那么这条 sampled tail 没有真正制造偏移。当前实现采用宽松但直接的处理：

- 重采若干次；
- 若仍然没有偏离 gold，则该样本该步跳过 KD supervision。

clean 模式没有 sampled tail，因此不做偏离判定，也不会因为没有 deviation 而跳过 KD。这里的核心原则是：**带 sampled tail 的样本应真实反映 student 自己自然走进偏移状态后的恢复能力**，而 clean 样本则提供普通 continuation anchor。

#### 5. 切回正常 decoding

一旦 sampled tail 结束，student 立刻切回正常 decoding 策略。
初版优先用：

- **greedy decoding**

此时 student 从当前 prefix

\[
\tilde y_{<\tau+m}
\]

继续 rollout，生成长度为 \(K\) 的后续 token：

\[
\hat y_{\tau+m:\tau+m+K-1}
\]

这里的语义是：sampled tail 只负责把 state 带到自然生成的当前状态；真正要训练 student continuation / recovery 能力的，是后面的正常 rollout 段。

#### 6. recovery rollout 直接作为 KD 段

初版不再单独引入“更长 rollout + 额外 loss mask”的机制。  
更直接的做法是把切回正常 decoding 后的 recovery rollout 长度直接定义为 \(K\)，并只对这 \(K\) 个 token 做蒸馏：

\[
t \in [\tau+m, \tau+m+K-1]
\]

也就是说：

- sampled tail 只负责构造当前 state；
- recovery 段负责继续或恢复；
- KD 的计算长度就等于 recovery rollout 的长度。

这样实现更简单，也更符合方法直觉：我们关心的就是“从当前 rollout state 出发，接下来这小段正常 rollout 的质量”。

### 训练时 teacher 的条件构造

在正常 decoding 阶段的任意位置 \( t \in [\tau+m, \tau+m+K-1] \)，student 实际看到：

\[
\tilde y^S_{<t} = [y_{<\tau}, \hat y^{\text{sampled}}_{\tau:\tau+m-1}, \hat y_{\tau+m:t-1}]
\]

teacher 则看到：

\[
\tilde y^T_{<t} =
\bigl[
x,\;
y,\;
y_{<\tau},\;
\hat y^{\text{sampled}}_{\tau:\tau+m-1},\;
\hat y_{\tau+m:t-1}
\bigr]
\]

其中：

- \( y \) 是完整 gold solution
- \( y_{<\tau} \) 是 gold prefix
- \( \hat y^{\text{sampled}}_{\tau:\tau+m-1} \) 是 student 自然生成的 sampled tail；clean 模式下这一项为空
- student 在切回正常 decoding 后的 rollout 历史直接拼接在 teacher prompt 末尾

因此 teacher 的 conditioning 是“完整正确解答 + student 当前真实 partial trajectory + student rollout 历史”，而不是“对同一 prefix 做 token patch”。

这里最重要的约束是：

- student rollout 只有一条，就是 student 自己生成出来的那条轨迹；
- teacher 不生成另一条 teacher rollout；
- teacher 只是在更强的条件上下文下，对这同一条 student rollout 提供蒸馏信号。

### 输入示例

下面给出一个简化示例。设原题与 gold solution 为：

```text
Problem: Solve 2x + 3 = 11.

Gold solution:
2x + 3 = 11, so 2x = 8, hence x = 4.
```

假设先取一段正确的 gold prefix：

```text
Problem: Solve 2x + 3 = 11.

2x + 3 = 11, so
```

然后 student 用 careless decoding 短暂 rollout 两个 token，恰好生成：

```text
 2x = 6
```

于是 student 进入了一个自然形成的局部错误状态。  
此时切回正常 decoding，并且 student 后续继续生成：

```text
 , hence x = 3. Therefore the answer is \boxed{3}.
```

则 student 实际输入示例为：

```text
Problem: Solve 2x + 3 = 11.

Please reason step by step, and put your final answer within \boxed{}.

2x + 3 = 11, so 2x = 6
```

teacher 输入示例为：

```text
Problem: Solve 2x + 3 = 11.

Here is a reference solution to this problem:
=== Reference Solution Begin ===
2x + 3 = 11, so 2x = 8, hence x = 4.
=== Reference Solution End ===

Use the reference solution only as private context for continuing the partial solution below. The marker "[recent sampled tail begins here]" indicates that the short tail after it may contain a local mathematical or wording inconsistency. Continue from exactly where the partial solution stops, in the style of a normal math solution. If the recent tail is inconsistent, continue in a way that restores mathematical coherence. Continue directly with the next math step.

Here is the partial solution to continue:
2x + 3 = 11, so [recent sampled tail begins here] 2x = 6 , hence x = 3. Therefore the answer is \boxed{3}.
```

这个 teacher 示例有意与原 `opsd` 路径保持较大重叠：

- 前半段仍然是 `Problem + Reference Solution`
- 中间仍然明确 reference solution 是 private context
- 后半段不再要求 teacher 独立重新推理，也不使用 XML/HTML-like 边界；只用一个低干扰 marker 标出 sampled tail 的开始

这里 teacher 没有自己的独立 rollout；它只是基于更强的上下文，对同一段 student rollout 提供 next-token supervision。

### 损失函数

不使用 CE，只保留蒸馏损失。当前实现使用 `loss_mode=jsd`，并用 `beta` 控制 forward KL、reverse KL 或 generalized JSD。在 student 切回正常 decoding 后的 recovery rollout 段上定义：

\[
L_{\mathrm{KD}}=
\frac{1}{K}
\sum_{t=\tau+m}^{\tau+m+K-1}
D_{\beta}(p_t, q_t)
\]

其中当前实现里的端点约定是：

- `beta=0`：forward KL，等价于 \(\mathrm{KL}(q_t \,\|\, p_t)\)
- `beta=1`：reverse KL，等价于 \(\mathrm{KL}(p_t \,\|\, q_t)\)
- \(0 < \beta < 1\)：generalized JSD

默认重要对照仍先用 `beta=0`，即 pure forward KL。

总损失即：

\[
L = L_{\mathrm{KD}}
\]

当前实现保留两个额外 loss 相关开关：

- `top_k_loss`：可选地只在 teacher top-k vocabulary 上做 JSD；
- `jsd_token_clip`：可选地裁剪 per-token divergence；当前 LinearOPSD 脚本显式设为 `0`，即关闭 clipping，因为 component-level clipping 会破坏分布蒸馏信号。

### 实现约束

这一版最大的实现变化是：局部错误前缀不再能在 data collator 内静态构造，因为它依赖一次真实 student rollout，并且需要在同一条样本内切换 decoding 策略。

因此在 `LinearOPSD` 的实现上，应遵循：

- 不新增新的 `conditioning_mode`
- 直接改现有 `linear_opsd`
- teacher prompt 组织方式尽量复用 / 参考现有 `opsd` 这条 prompt 风格
- careless rollout 的生成逻辑应放在更靠近 trainer / rollout 的位置，而不是继续写成纯静态 collator 逻辑
- teacher trace 中只在 sampled tail 前加入低干扰 marker；默认不用 `<careless>` / `<recovery>` 这类 XML/HTML-like 标记
- 若 sampled tail 没有真正偏离 gold，则允许重采；若仍未偏离，则该样本该步跳过 KD
- student 和 teacher 的 Qwen chat-template thinking mode 分别由 `linear_opsd_student_enable_thinking` 和 `linear_opsd_teacher_enable_thinking` 控制，默认都关闭，便于先跑 non-thinking sanity check
- 不需要为了兼容旧实现而保留手工 corruption point、replacement token、`<corrupt>` 标记等路径

### 训练细节

- teacher 策略可在 dynamic / fixed / EMA 之间切换
- 每条样本每步先保留一段 gold prefix，再按 clean/mild/hard mixture 决定是否生成 sampled tail
- sampled tail 以更高 temperature / top-p / top-k 采样为主，用来制造自然偏移状态
- 若 sampled tail 与 gold 完全一致，则重采或跳过该样本该步监督；clean 样本不做 deviation check
- sampled tail 结束后，student 立刻切回正常 decoding
- teacher trace 写成 `gold prefix + optional sampled-tail marker + optional sampled tail + recovery rollout`
- student 正常 rollout 解码初版优先使用 greedy
- teacher 与 student 共享同一条 student rollout 轨迹
- recovery rollout 的长度 \( K \) 直接等于蒸馏长度
- `LinearOPSD` 路径仍以 full-vocabulary logit distillation 为主
- 当前脚本默认关闭 `jsd_token_clip`

### 评测

#### 1. 主评测：error-correction robustness

在 held-out math 样本上固定相同的 `gold prefix + optional sampled tail + normal recovery` 协议，报告：

- clean prefix 下的解题准确率
- sampled-tail prefix 下的解题准确率
- recovery gap

\[
\text{recovery gap} = \text{clean acc} - \text{sampled-tail acc}
\]

- rollout 段平均

\[
\mathrm{KL}(q_t \,\|\, p_t), \qquad \mathrm{KL}(p_t \,\|\, q_t)
\]

以及混合损失

\[
L_{\mathrm{KD}}
\]

这一组指标仍是主指标，用来衡量模型是否更会从局部错误 CoT 中恢复。

#### 2. benchmark

- **GSM8K**
- **MATH-500**
- **MMLU-Pro**
- **IFEval**

其中前两者看数学泛化，后两者看 general capability / instruction-following 是否被明显破坏。

### 最小实验矩阵

主实验先只跑以下几组：

- **A**：`beta=0`，`rollout_decoding=greedy`，`jsd_token_clip=0`
- **B**：`beta=0`，`rollout_decoding=sample`，`jsd_token_clip=0`
- **C**：`beta=0.5`，`rollout_decoding=greedy`，`jsd_token_clip=0`
- **D**：`beta=1`，`rollout_decoding=greedy`，`jsd_token_clip=0`

然后再做以下小规模消融：

- clean/mild/hard mixture 比例
- sampled-tail 长度：mild \(m \in \{4, 8, 16\}\)，hard \(m \in \{8, 16, 24\}\)
- recovery rollout 长度：\( K \in \{8, 16, 128\} \)
- careless decoding 强度：低温 vs 高温 / 更小 top-p
- teacher 策略：dynamic vs EMA vs fixed
- student / teacher thinking mode：默认都关闭，必要时分别打开

### 与 5.md 的主要区别

相较于 `5.md`，这一版的变化是：

- teacher 不再只拿 clean local patch，而是拿完整 gold solution
- teacher prompt 中显式加入 student 的 optional sampled tail 与 recovery rollout
- 不再默认用 `<careless>` 和 `<recovery>` 这类 XML/HTML-like marker，而是只在 sampled tail 前使用低干扰 marker
- corruption 不再是随机 span 替换或基于熵的手工 token 替换，而是 student 自己的 optional sampled-tail rollout
- 方法目标从“local patch supervision”进一步转向“natural sampled-state continuation / recovery”
